
باسلام و احترام

برای اینکه UI جدید را به Jira متصل کنیم، فعلاً ۳ API آماده شده که می‌توانی از آن‌ها استفاده کنی و تست بگیری.

-1 دریافت اطلاعات کاربر جاری
Method: GET
URL: https://jira.vosouq.me/rest/scriptrunner/latest/custom/me
کاربرد:
•	دریافت اطلاعات کاربر لاگین شده
•	نمایش نام، نام کاربری و ایمیل در UI
نمونه خروجی:
{
    "username":"ra.atyabi",
    "displayName":"Rasol Atyabi",
    "email":""
}

-2 دریافت کاربران
Method: GET
URL: https://jira.vosouq.me/rest/scriptrunner/latest/custom/users
کاربرد:
•	دریافت اطلاعات کاربران جهت استفاده در فرم‌ها (در مراحل بعدی قابلیت جستجو هم اضافه می‌شود.)

-3 دریافت Request Type ها و فیلدهای هر فرم
Method: GET
URL: https://jira.vosouq.me/rest/scriptrunner/latest/custom/requestTypes

کاربرد:
•	دریافت لیست فرم‌های قابل ثبت
•	دریافت فیلدهای مربوط به هر Request Type
•	ساخت فرم به صورت داینامیک در UI
نمونه خروجی:
[
  {
    "name":"پشتیبانی نرم افزار",
    "fields":[
      {
        "id":"summary",
        "name":"موضوع درخواست"
      },
      {
        "id":"customfield_10302",
        "name":"محصول"
      }
    ]
  }
]

برای تست API ها
تمام API های بالا با Method GET هستند و می‌توانی از طریق مرورگر یا Postman آن‌ها را تست کنی.

مرحله بعد) API شماره 4(
الان در حال آماده‌سازی API ثبت درخواست هستم.برای تکمیل این API لازم است دقیقاً بدانم UI هنگام فشردن دکمه ثبت چه اطلاعاتی برای Backend ارسال خواهد کرد.لطفاً یک نمونه کامل از Request Body که قرار است ارسال شود برای من بفرست.
مثلاً چیزی شبیه:
{
    "requestType":"پشتیبانی نرم افزار",
    "summary":"مشکل VPN",
    "description":"از صبح وصل نمی‌شود",
    "priority":"High",
    "duedate":"2026-06-30",
    "customfield_10302":"Jira"
}
یا اگر ساختار دیگری مدنظر داری، همان ساختار را ارسال کن.
همچنین لطفاً مشخص کن فایل پیوست را به چه روشی ارسال می‌کنی:
•	multipart/form-data
•	Base64
•	یا روش دیگری
بر اساس همین ساختار، API شماره 4 را پیاده‌سازی می‌کنم تا بعد از کلیک روی Submit، Request به صورت مستقیم در پروژه ITSM ایجاد شود.
